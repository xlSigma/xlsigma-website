## How to Work on This Site

### The Golden Rule
**Always edit `xlsigma-files.ps1`, never edit `.tsx` files directly.**
Running the script overwrites all site files. Manual edits to `.tsx` files will be lost.

### Workflow Every Session
1. cd C:\Users\andre\dev\xlsigma-website
2. git pull                          # sync with GitHub
3. Edit xlsigma-files.ps1 in VS Code
4. .\xlsigma-files.ps1               # regenerate site files
5. npm run dev                        # preview at http://localhost:3000
6. git add .
7. git commit -m "describe change"
8. git push                           # auto-deploys to xlsigma.com via Vercel

### Key Files
| File | Purpose |
|------|---------|
| `xlsigma-files.ps1` | **Main script** — generates all site `.tsx` files |
| `xlsigma-contact-checkboxes.ps1` | Patches the contact page checkboxes |
| `HANDOFF.md` | Full architecture, stack, DNS, deployment reference |
| `app/page.tsx` | Homepage (generated by script) |
| `app/capabilities/page.tsx` | Capabilities page (generated) |
| `app/government-contracting/page.tsx` | Gov contracting page (generated) |
| `app/contact/page.tsx` | Contact page (generated) |
| `app/components/NavBar.tsx` | Shared navigation (generated) |
| `app/components/Footer.tsx` | Shared footer (generated) |

### PowerShell Script Rules
- Scripts use **single-quoted here-strings** (`@'...'@`) — never double-quoted
- Never use `Write-Output` for file content
- Node.js is used internally for file writes to avoid Windows-1252 encoding issues

### Tools Andres Has
- **VS Code** — preferred editor (`code filename` from terminal)
- **Notepad++** — alternate editor
- **Notepad** — built-in fallback
- **Windows PowerShell 5.x** (the blue one) — use for running scripts
